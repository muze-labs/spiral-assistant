# Roadmap-Cycle Planning Prompt

Use this prompt to plan the next spiral cycle.

```text
Use the Spiral Assistant repository as background.

Help plan the next roadmap-driven spiral cycle.

Inputs:
- current project brief;
- roadmap;
- maturity matrix;
- risk register;
- boundary map;
- abstraction review;
- cycle log, if available.

Produce a cycle plan with:

1. cycle name;
2. roadmap capability being validated;
3. main risk being reduced;
4. selected quality metrics;
5. target scores;
6. non-regression metrics;
7. explicit non-goals;
8. evidence needed;
9. accepted debt, if any;
10. AI usage boundaries;
11. conditions that would pause or alter the roadmap.

Make sure the cycle is small enough to produce evidence.

Prefer clarity over completeness. Do not harden poor abstractions.

End with a cycle decision checklist:

- What question will this cycle answer?
- What evidence will be accepted?
- What complexity is being added?
- What complexity should be removed or avoided?
- What would cause a pause before the next cycle?
```
