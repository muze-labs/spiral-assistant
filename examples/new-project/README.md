# Example: New Project Intake

This example shows the kind of output the assistant should produce for a new project.

It is intentionally generic. Replace it with real Muze examples once the workflow has been tested.

## Project idea

A small browser-based tool that lets non-professional users collect notes, organize them visually, and publish a simple webpage.

## Expected first assistant outputs

- `MUZE_PROJECT_BRIEF.md`;
- `MUZE_ROADMAP.md`;
- `MUZE_RISK_REGISTER.md`;
- `MUZE_MATURITY.md`;
- one recommended first spiral cycle.

## Example first cycle

Cycle: Usable interaction prototype

Main risk:

> Users may not understand the organize-and-publish flow.

Quality focus:

- usability;
- visual clarity;
- feedback value;
- conceptual simplicity.

Non-goals:

- public hosting;
- authentication;
- production persistence;
- collaboration;
- full schema design.

Evidence:

- clickable/usable prototype;
- one or two user walkthroughs;
- notes on confusing vocabulary;
- list of abstraction problems discovered.

Pause conditions:

- users cannot explain the organize-and-publish flow after trying it;
- the vocabulary requires repeated explanation;
- the data model needs special cases before the interaction is validated.
